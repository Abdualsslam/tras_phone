# ملحق (أ) نطاق العمل التفصيلي - مشروع TRAS Phone

> نسخة مبنية على فحص فعلي لهيكلة المشروع الحالية (Mobile + Admin + Backend + Deployment + Docs).

## A1) مكونات المشروع

1) تطبيق العميل (Flutter Mobile App) - Android/iOS
2) لوحة تحكم إدارية (React + TypeScript + Vite)
3) Backend API (NestJS) + MongoDB + Redis
4) خدمات مساندة وتشغيل (Uploads, Jobs, Cache, Audit, Integrations)
5) إعدادات النشر بالحاويات (Docker + Nginx + Health Checks)
6) وثائق تشغيل وتكامل واختبارات يدوية (docs/)

## A2) الأدوار والصلاحيات

- عميل (Customer/B2B): تسجيل، تصفح، شراء، مدفوعات، طلبات، مرتجعات، محفظة، دعم، ملف شخصي.
- مدير/موظف إداري (Admin): إدارة التشغيل اليومي عبر اللوحة.
- مدير نظام (Super Admin): صلاحيات حساسة إضافية (حذفات/إعدادات عليا/قوالب متقدمة).
- نظام صلاحيات ديناميكي (RBAC): إدارة Roles + Permissions من داخل النظام.
- مصفوفة صلاحيات موسعة: 95 صلاحية موزعة على الوحدات (Users, Customers, Products, Orders, Inventory, Promotions, Support, Reports, System...).

## A3) تطبيق العميل - الشاشات والوظائف

### 1) المصادقة والحساب

- Splash + Onboarding.
- تسجيل جديد، تسجيل دخول، دخول ببصمة، نسيان كلمة المرور، OTP، إعادة تعيين كلمة المرور.
- إدارة الجلسات وتحديث FCM Token من خلال تكامل المصادقة.

### 2) الصفحة الرئيسية والتصفح

- Home مع بانرات، أقسام، منتجات مميزة، وإشعارات.
- تدفق كتالوج متكامل: فئات -> علامات تجارية -> أجهزة -> منتجات.
- بحث عادي + بحث متقدم + سجل بحث + نتائج بحث.

### 3) المنتجات والتفاعل

- قائمة منتجات مع فرز/فلترة متعددة.
- تفاصيل منتج، منتجات على العرض، منتجات حسب الفئة/العلامة/الجهاز.
- تقييمات المنتج: عرض التقييمات، كتابة تقييم، مراجعاتي، التقييمات المعلقة.
- مفضلة + تنبيهات توفر/سعر (Stock Alerts).

### 4) السلة والشراء

- سلة محلية/مزامنة مع الخادم، تعديل كميات، تطبيق كوبون.
- Checkout Session متكامل (عناوين + طرق الدفع + كوبون + بيانات العميل).
- اختيار طريقة الدفع، اختيار عنوان التوصيل، تأكيد الطلب.

### 5) الطلبات وما بعد البيع

- قائمة الطلبات، تفاصيل الطلب، تتبع الطلب.
- رفع إيصال التحويل البنكي، عرض/تحميل الفاتورة.
- المرتجعات: اختيار العناصر، إنشاء طلب إرجاع، قائمة المرتجعات، تفاصيل الإرجاع.

### 6) المحفظة والولاء

- رصيد المحفظة، سجل العمليات.
- نقاط الولاء، مستويات الولاء، معاملات الولاء.
- الإحالة (Referral).

### 7) الدعم والإشعارات والمحتوى

- إشعارات: قائمة + تفاصيل + تعليم كمقروء.
- الدعم: تذاكر، تفاصيل، محادثة التذكرة، إنشاء تذكرة، Live Chat.
- محتوى تعليمي: فئات، قوائم، تفاصيل، بحث، مفضلة، محتوى مرتبط بالمنتج.
- صفحات ثابتة: الشروط، الخصوصية، من نحن، FAQ.

### 8) الملف الشخصي والإعدادات

- الملف الشخصي، تعديل البيانات، تغيير كلمة المرور.
- إدارة العناوين (إضافة/تعديل) + اختيار موقع على الخريطة.
- إعدادات اللغة والإشعارات والثيم.

## A4) لوحة التحكم - الصفحات والوظائف

### 1) التشغيل اليومي

- Dashboard: مؤشرات تشغيلية، مبيعات، طلبات حديثة، مخزون منخفض، إجراءات معلقة.
- Analytics: رسوم وتقارير مبيعات/طلبات/عملاء/بحث/منتجات.
- Orders + Order Details: إدارة حالات الطلب، الشحن، الملاحظات، الفواتير، التحقق من الحوالات.
- Returns: إدارة دورة المرتجعات والفحص والاسترداد.

### 2) الكتالوج والتسعير

- Products: CRUD منتجات، إدارة الصور، التوافق مع الأجهزة، مراجعات، تنبيهات مخزون.
- Categories + Catalog: إدارة الفئات/العلامات/الأجهزة/أنواع الجودة.
- Price Levels: مستويات تسعير العملاء.
- Promotions: إدارة العروض والكوبونات والإحصائيات.

### 3) الإمداد والمخزون

- Inventory: مستودعات، مواقع تخزين، رصيد المخزون، حجوزات، إحصائيات.
- Suppliers: موردين ومنتجات المورد والمدفوعات.
- Purchase Orders: إنشاء/تحديث/استلام أوامر الشراء.

### 4) العملاء والإدارة

- Customers: إدارة العملاء، الموافقات/الرفض، الشرائح، الائتمان.
- Admins + Roles: إدارة المستخدمين الإداريين والأدوار والصلاحيات.
- Password Reset Requests: متابعة طلبات إعادة تعيين كلمة المرور للعملاء.
- Wallet: إدارة رصيد العملاء ونقاط الولاء والمعاملات.

### 5) المحتوى والدعم والحوكمة

- Content: FAQ، صفحات، بانرات، سلايدر، Testimonials.
- Educational Content: فئات ومحتوى تعليمي مع نشر واستهداف سياقي.
- Notifications: قوالب، حملات، إرسال مباشر.
- Support + Live Chat: إدارة التذاكر، القوالب الجاهزة، المحادثات الفورية، تقارير الدعم.
- Audit Logs: سجلات تدقيق، نشاط الإداريين، سجل الدخول، محاولات مشبوهة.
- Settings: إعدادات المتجر، الشحن، الضرائب، الدول/المدن، طرق الدفع، الإشعارات، إعدادات النظام.

## A5) نطاق Backend / API

### 1) الوحدات الأساسية

- Auth, Users, Customers, Admins/Roles/Permissions.
- Products, Categories, Catalog, Pricing.
- Cart, Checkout, Orders, Payments, Invoices.
- Inventory, Suppliers, Purchase Orders.
- Returns, Wallet, Loyalty.
- Notifications, Support (Tickets + Chat + Reports).
- Content + Educational Content.
- Settings, Locations, Analytics, Audit, Uploads.

### 2) خصائص API الجوهرية

- REST APIs موحدة الرسائل (AR/EN) مع Pagination في القوائم.
- Swagger على `/api/docs`.
- حماية endpoints عبر JWT + Guards + RBAC.
- تسجيل نشاط إداري (Audit interceptor) وتاريخ الدخول.

## A6) البيانات والتكاملات

- قاعدة البيانات: MongoDB (Schemas متعددة تغطي كل الوحدات التجارية).
- الكاش: Redis (جلسات/أداء/طبقات دعم).
- التخزين: S3-compatible uploads (صور/مرفقات).
- إشعارات Push: FCM.
- قنوات مراسلة/بريد: دعم إعدادات SMS/Email عبر متغيرات البيئة.
- دفع/شحن: متغيرات بيئة مهيأة لتكاملات خارجية (HyperPay/Moyasar, SMSA/Aramex) إضافة إلى تدفقات الدفع الداخلية.
- خرائط: Google Maps في تطبيق الموبايل.

## A7) المتطلبات غير الوظيفية (NFR)

- الأمان: JWT، تشفير كلمات المرور، Validation صارم، Helmet، CORS، Throttling افتراضي 100 طلب/دقيقة.
- الأداء: Compression، Redis cache، Pagination، بحث متقدم/اقتراحات.
- الاعتمادية: Health checks للحاويات، Restart policy، فصل الخدمات (Backend/Admin/Redis).
- القابلية للتوسع: بنية Modular في NestJS + Monorepo + Modules مستقلة.
- الرصد والتدقيق: Logging interceptor، Audit logs، نشاط إداري تفصيلي.
- التدويل: دعم عربي/إنجليزي في API ورسائل الواجهة (Mobile/Admin).

## A8) التشغيل والنشر

- تشغيل محلي منفصل لكل مكون (`backend`, `admin`, `mobile`).
- نشر حاويات عبر `docker-compose.yml` على مستوى المشروع.
- Admin يُبنى كملفات static ويُخدم عبر Nginx.
- Backend وRedis معزولان شبكياً مع فحوصات جاهزية.
- يدعم نمط بيئات متعددة عبر ملفات `.env` (Development/Production وقابلية تجهيز Staging بنفس النمط).

## A9) مخرجات التسليم

1) الكود المصدري الكامل (Mobile/Admin/Backend).
2) ملفات البيئة والنشر (Dockerfiles + Compose + Nginx configs).
3) وثائق API وتشغيل وتكامل (`README`, `API_GUIDE.md`, `docs/`).
4) وثائق فحص/تكامل Flutter وقوائم تحقق يدوية داخل `docs/flutter-integration/`.
5) سكربتات Seed/Migration/صيانة بيانات في Backend.

## A10) خارج النطاق الحالي (Out of Scope)

- ERP خارجي أو نظام محاسبي كامل متكامل (غير ظاهر كتنفيذ مباشر في المشروع).
- POS/نقاط بيع فعلية داخلية.
- واجهة متجر ويب للعملاء (Customer Web Storefront) - المتاح حالياً: Mobile App + Admin Panel.
- تكاملات طرف ثالث غير المفعّلة فعلياً في البيئة (حتى يتم توفير المفاتيح والإعداد التشغيلي).

## A11) ملاحظات تنفيذية

- المشروع متعدد التطبيقات ومُهيأ للإنتاج، مع تركيز واضح على B2B لقطع غيار الهواتف.
- نطاق العمل الحالي يتجاوز الأساسيات التقليدية للتجارة الإلكترونية بإضافة:
  - دعم فني متقدم (Tickets + Live Chat + تقارير).
  - نظام محفظة وولاء.
  - نظام محتوى تعليمي سياقي مرتبط بالمنتجات.
  - تدقيق أمني وتشغيلي موسع.
